1. File System Interfaces

1.1 List Files or Folders

  • Endpoint Path: /api/fs/list
  • Request Body (JSON): 
    {
  "path": "String or Array", // Target path
  "type": 1, // 1: Scan folder, 2: Scan file
  "exts": "jpg,png" // (Optional) Only valid when type is 2, filters file extensions separated by commas
}
  • Response Status Codes:
    • 200 OK: Successfully returned file or folder list data.

1.2 Get File Content

  • Endpoint Path: /api/fs/get
  • Request Body (JSON): 
    {
  "path": "String or Array" // Relative path of the file
}
  • Response Status Codes:
    • 200 OK: Success, returns file binary stream (Buffer).
    • 404 Not Found: File does not exist.
    • 500 Internal Server Error: Failed to read file.

1.3 Write/Upload File

  • Endpoint Path: /api/fs/set
  • Request Headers:
    • File-Path: Target save directory
    • File-Name: Target file name (If not provided, the system will automatically generate noname_timestamp.bin)
  • Request Body:
    • Raw binary data of the file (Binary).
  • Response Status Codes:
    • 201 Created: File written successfully.
    • 500 Internal Server Error: Failed to write file.

1.4 Delete Files

  • Endpoint Path: /api/fs/del
  • Request Body (JSON Array): 
    [
  "path/to/file1.txt",
  "path/to/file2.png"
]
  • Response Status Codes:
    • 204 No Content: Batch deletion successful.
    • 404 Not Found: Non-existent files exist in the files to be deleted.
    • 500 Internal Server Error: Underlying error occurred during deletion.

2. Configuration Management Interfaces (Settings)

Permission Notice: Configuration management related interfaces restrict the access source, allowing only local requests whose Host is 127.0.0.1:{port} or localhost:{port} to access.

2.1 Get Current Configuration

  • Endpoint Path: /api/settings/get
  • Request Headers:
    • Requires a correct Host header.
  • Response Status Codes:
    • 200 OK: Returns global configuration object (includes parameters like sanjuuni, input, output, net, etc.).
    • 403 Forbidden: Non-local request, access denied.

2.2 Modify/Write Configuration

  • Endpoint Path: /api/settings/set
  • Request Headers:
    • Requires a correct Host header.
  • Request Body (JSON): 
    {
  // Key-value pairs of configuration items to be modified
}
  • Response Status Codes:
    • 201 Created: Configuration modified successfully.
    • 403 Forbidden: Non-local request, access denied.
    • 500 Internal Server Error: Configuration parameter validation failed or write error.

3. Task/Job Interfaces (Job)

3.1 Get Current Job Status

  • Endpoint Path: /api/job/status
  • Response Body (JSON): 
    {
  "inRunning": false, // Whether it is currently running
  "inError": false, // Whether an error occurred
  "time": 0, // Execution elapsed time
  "wait": [], // Waiting queue
  "processing": [], // Processing queue
  "finish": [], // Finished queue
  "ignore": [] // Ignored queue
}
  • Response Status Codes:
    • 200 OK: Successfully returned status.

3.2 Start Executing Task

  • Endpoint Path: /api/job/start
  • Request Body (JSON Array): 
    [
  // List of task/image data to be processed
]
  • Response Status Codes:
    • 201 Created: Task successfully triggered and moved to background asynchronous execution (returns 201 directly if an empty array is passed).
    • 400 Bad Request: Request body is not a valid array.
    • 503 Service Unavailable: An existing task is currently running, cannot start repeatedly.

3.3 Clear Error Status

  • Endpoint Path: /api/job/clearError
  • Description: Resets inError in the job status to false.
  • Response Status Codes:
    • 201 Created: Reset successful.
